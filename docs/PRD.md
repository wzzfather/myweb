# PRD - 顶栏个人简介区域集成网站 QR 码

> 文档版本：v0.1（初稿）
> 编写日期：2026-06-02
> 编写人：产品经理

---

## 1. 需求概述

### 1.1 背景说明
当前个人简历网站（`https://wazzfather.github.io/myweb/`）通过 GitHub Pages 部署，但页面本身缺少"扫码即访问"的入口。在简历分享、面试投递、线下交流等场景下，他人若已在 PC 上浏览网页，反而难以将站点快速转到手机；反之在手机上看到的人也难以复制链接。

### 1.2 目标用户
- **访问者**：通过 PC 浏览器或线下场景（如简历、海报）首次接触本站的 HR、面试官、合作方、同行朋友。
- **站点主人（魏子政本人）**：希望降低"分享 → 复制 URL → 在另一台设备打开"的链路成本。

### 1.3 核心价值
- 让任何访客用手机扫码即可在 1 秒内将本站转到移动端打开/收藏/转发；
- 让站点主人无需借助第三方二维码生成器，即可在自托管页面提供"标准 QR 码"。

## 2. 项目概览

### 2.1 技术栈一句话总结
**纯静态站点**：原生 HTML5 + 原生 CSS3 + 原生 JavaScript（ES5/ES6），无任何前端框架、无构建工具、无包管理器，外部依赖仅 Font Awesome 6（CDN），通过 GitHub Pages 部署。

### 2.2 顶栏与个人简介区域现状
- **顶栏（导航栏）**：`index.html:14-31` —— `<nav class="navbar">`，内部仅含 `.nav-menu` 列表（关于我 / 工作经历 / …）与移动端汉堡按钮 `.hamburger`。
- **个人简介区域（Hero）**：`index.html:34-51` —— `<header class="hero" id="about">`，依次包含：
  - `.profile-img` 圆形头像（`index.html:36-38`）
  - `<h1>` 姓名（`index.html:39`）
  - `.title` 职位（`index.html:40`）
  - `.description` 自我介绍段落（`index.html:41-43`）
  - `.hero-buttons` 操作按钮组（`index.html:44-49`），现含「联系我」「下载简历」两个 `.btn`
- **样式约定**：`.hero` 主色为深绿渐变（`css/style.css:81-88`），白色前景；按钮统一样式定义于 `css/style.css:156-190`（`.btn` / `.btn.primary` / `.btn.secondary`）。
- **JS 入口**：`js/script.js`，使用 `DOMContentLoaded` 注册各类交互（已有装饰图片、年龄动态计算等模块）。

### 2.3 关键约束
- 无构建系统 → 引入第三方库必须通过 **CDN `<script>` 标签** 或本地 vendor 文件。
- 站点为静态托管 → 必须 **100% 前端生成 QR 码**，不可依赖后端服务。
- 当前站点 URL 在仓库中已硬编码于 `index.html:435`：`https://wazzfather.github.io/myweb/`，可作为 QR 内容来源。

## 3. 需求理解

### 3.1 用户故事
- **作为访问者**，我希望在首页顶部一眼能看到一个二维码，以便用手机扫码立即打开/收藏/转发本站。
- **作为访问者**，我希望可以一键复制网站 URL，省去手动选中浏览器地址栏的麻烦。
- **作为访问者**，在移动端访问时，我也希望能看到二维码（便于"出示给他人扫"或保存图片）。
- **作为站点主人**，我希望二维码的样式与现有深绿/白色 Hero 区域视觉风格统一，不破坏首页设计。

### 3.2 核心场景
| 场景 | 触发方式 | 期望结果 |
|---|---|---|
| PC 端首次访问 | 滚动到 Hero 区，看到"扫码访问"按钮 | 点击后在弹窗中展示大尺寸 QR 码 |
| PC 端复制链接 | 弹窗内点击"复制链接"按钮 | 链接写入剪贴板，给出成功提示 |
| 移动端访问 | 直接看到 QR 码缩略图 / 按钮 | 可长按保存或转发 |
| 线下分享 | 投影/截图首页 | 截图中可见二维码或显眼入口 |

## 4. 功能方案

### 4.1 主方案（推荐）：Hero 区按钮 + 中央弹窗（Modal）

**交互流程**
1. 在 `.hero-buttons`（`index.html:44-49`）中追加一个新按钮 `扫码访问`（图标使用 Font Awesome 的 `fa-qrcode`，与现有按钮一致样式 `.btn.secondary`）。
2. 用户点击 → 触发 `js/script.js` 中的新函数 `openQrModal()`，动态生成/打开一个全屏遮罩 + 居中卡片：
   - 卡片内含：QR 码图片（240×240）、URL 文本（`https://wazzfather.github.io/myweb/`）、「复制链接」「下载二维码」「关闭」三个操作按钮。
3. QR 码在弹窗**首次打开时**才生成（懒加载，避免首屏多余开销）。
4. 关闭方式：点击关闭按钮 / 点击遮罩 / 按 `Esc`。

**UI 位置**
- 入口：Hero 区按钮组第三个按钮（`index.html:44` 区域）。
- 弹窗：全屏遮罩 + 居中卡片，与现有深绿主题色协调（白色卡片 + 主品牌色按钮）。
- 移动端（`@media (max-width: 768px)`，参考 `css/style.css:575-637`）：弹窗自适应宽度 90vw，QR 码缩小至 180×180。

**依赖库（CDN，无构建）**
- **推荐 `qrcodejs`**（davidshimjs/qrcodejs）：
  - 纯 JavaScript、无依赖、UMD 直接通过 `<script>` 引入；
  - CDN：`https://cdn.jsdelivr.net/npm/qrcodejs@1.0.0/qrcode.min.js`（约 18 KB）；
  - API 极简：`new QRCode(element, { text, width, height, colorDark, colorLight })`；
  - 输出为 `<img>` + `<canvas>`，移动端长按可保存，便于"下载二维码"功能实现（直接对 canvas 执行 `toDataURL`）。
- **备选 `qrcode-generator`**（Kazuhiko Arase）：同等定位，输出 table/img；若主库 CDN 不稳定可切换。

**URL 来源策略**
- 默认值：`https://wazzfather.github.io/myweb/`（与 `index.html:435` 一致）。
- 运行时优先级：`window.location.href`（自动适配自定义域名 / 镜像站点 / 本地调试），仅在 `location.href` 取不到时回退到默认值。
  - 好处：未来若更换域名 / 部署到 Cloudflare Pages 等，无需改代码。

### 4.2 备选方案：始终在 Hero 右侧固定展示小尺寸 QR 码
- 在 `.hero-content` 旁加一个 100×100 的 QR 码常驻显示。
- 优点：免点击、截图即可用。
- 缺点：占用 Hero 区宝贵空间，与现有居中布局冲突，移动端体验差。
- **结论**：作为可选增强项（P2），暂不采用。

### 4.3 功能优先级
| 功能点 | 优先级 |
|---|---|
| Hero 区"扫码访问"按钮 | **P0** |
| 弹窗内 QR 码展示（含 URL 文本） | **P0** |
| 复制链接到剪贴板 | **P0** |
| 弹窗关闭（关闭按钮 / 遮罩点击 / Esc） | **P0** |
| 下载二维码为 PNG | **P1** |
| 移动端响应式适配 | **P0** |
| 常驻侧边小 QR 码（备选方案） | **P2** |

## 5. 技术可行性评估

### 5.1 推荐的 QR 码库及原因
**`qrcodejs`（davidshimjs/qrcodejs）**

| 评估项 | 结果 |
|---|---|
| 是否兼容当前技术栈 | ✅ 纯静态 HTML/JS，CDN 引入即可，零打包 |
| 包体积 | ~18 KB（gzip 后约 6 KB） |
| 输出格式 | `<img>` + `<canvas>`，便于下载、长按保存 |
| 浏览器兼容 | IE6+ / 所有现代浏览器 / 移动端 Safari/Chrome |
| 维护状态 | 核心代码稳定，无需频繁更新 |
| 学习成本 | 单一构造函数，5 分钟可上手 |

**结论**：与本项目"无构建系统 + 极简架构"完美契合，无需引入 npm / 打包器。

### 5.4 难度等级
**🟢 低（Low）**

理由：
- 改动局限于 3 个静态文件，无后端、无构建、无框架升级；
- QR 码库为业界通用方案，无需自研算法；
- 不涉及现有功能改造，无回归风险。

### 5.5 风险与注意事项
| 风险 | 等级 | 缓解措施 |
|---|---|---|
| CDN 失效导致 QR 码不渲染 | 低 | ① 选择 jsDelivr CDN（多镜像冗余）；② 代码中加 `window.QRCode` 判断，缺失时降级为"复制链接"按钮，不阻塞主流程 |
| QR 码图片在深绿背景下识别率下降 | 低 | QR 码强制使用 `#ffffff` 背景 + `#000000` 前景（高对比），仅外层卡片使用品牌色 |
| 移动端弹窗遮挡、关闭手势不顺 | 低 | 弹窗点击遮罩即可关闭；移动端宽度限制 90vw，并支持 `Esc` 与显式关闭按钮 |
| URL 在本地预览（`file://` 或 `localhost`）时与正式 URL 不一致 | 低 | 优先取 `window.location.href`，仅在异常时回退默认值（见 §4.1） |
| 浏览器剪贴板 API 兼容性 | 低 | 优先 `navigator.clipboard.writeText`，老浏览器降级为 `document.execCommand('copy')` |

## 7. 验收标准

### 7.1 功能验收清单
- [ ] **入口可见**：首页 Hero 区（`#about`）"联系我 / 下载简历"按钮旁出现第三个按钮「扫码访问」，文案与图标（`fa-qrcode`）正常显示。
- [ ] **QR 码生成正确**：点击按钮后，弹窗内 QR 码扫描后跳转至 `https://wazzfather.github.io/myweb/`（或当前 `window.location.href`），使用微信、支付宝、iOS 相机、Chrome 扫描均能识别。
- [ ] **复制链接可用**：点击「复制链接」后剪贴板内容为目标 URL，并在页面给出可见的成功提示（Toast / inline 文案），PC Chrome、Safari、移动端均验证通过。
- [ ] **下载二维码可用**：点击「下载图片」后，本地得到一张 ≥ 240×240 的 PNG 文件，文件名符合预期，无破损。
- [ ] **多种关闭方式**：① 点击「关闭」按钮；② 点击弹窗外部遮罩；③ 按 `Esc` 键 —— 三者均能关闭弹窗并释放遮罩。
- [ ] **响应式适配**：在 375px（iPhone SE）、768px（iPad）、1440px（PC）三档宽度下，弹窗不溢出、QR 码完整可见、按钮可点击。
- [ ] **懒加载**：弹窗未打开时，QR 库的 `<script>` 已加载但 QR 码 DOM 未渲染（首屏 DOM 节点数增量 ≤ 5）。
- [ ] **降级容错**：断网或 QR 库 CDN 失败时，按钮点击仍能弹出"复制链接"功能，不抛出 JS 错误。
- [ ] **无回归**：现有「联系我」「下载简历」按钮、导航锚点、装饰图片动画、年龄动态计算等已有功能均正常工作。

### 7.2 非功能验收
- [ ] **性能**：首屏 Lighthouse Performance 分数下降 ≤ 2 分（CDN 仅 ~18 KB，影响极小）。
- [ ] **视觉一致性**：弹窗配色（深绿主色 + 白色卡片 + 圆角）与现有 Hero 区按钮 / 卡片风格一致，无视觉割裂。
- [ ] **可访问性**：弹窗具备 `role="dialog"`、`aria-modal="true"`、`aria-labelledby`，关闭按钮含 `aria-label`，键盘 Tab 顺序正确。
